1 /*
2  * @file evaluation.hh
3  * @author Nick Weber
4  * @date Apr 29, 2018
5  * @brief This singleton class is used for evaluating runtime and information retrieval performance.
6  *
7  * @section DESCRIPTION
8  * This class consists of several nested classes not visible to the outside. The nested classes all
9  * handle different functionalities of evaluation. The nested class IR_PerformanceManager provides
10  * several IR evaluation methods such as accuracy, F-measure, MAP and DCG. For more details, read
11  * the description of the nested class. Users of this class do not have to worry about the internal
12  * details. A simple call to start/stop measures the runtime performance between the start and stop
13  * call. To Evaluate IR performance, a call to evalIR is enough to evaluate the ranking with all
14  * implemented IR evaluation techniques (as named above).
15  *
16  * @section USE
17  * 1. By calling Evaluation::getInstance(), the caller gets access to the evaluation instance.
18  * 2. Calling start on the evaluation object, starts run time measurement. Two arguments have to be
19  * provided: the enum value of the current IR mode to measure and a query name
20  * 3. Calling stop on the evaluation object, stops the runtime measurement and persists the
21  * measurement results in an internal data structure
22  * 4. Calling evalIR with the same arguments as start plus a ranking will calculate all IR evaluation
23  * methods and persist them in an internal data structure
24  * 5. A call to constructJSON will create the JSON object which can be used to parse the results etc..
25  */
